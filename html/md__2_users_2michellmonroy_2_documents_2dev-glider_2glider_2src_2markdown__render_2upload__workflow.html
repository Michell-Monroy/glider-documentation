Google Drive Submodule

It receives a google drive link, downloads reports in a temporal folder to applies the Cleaning Procedure

Workflow

How to create venv dev

  1. Create a new virtual env (if you have created this venv pass to step 4)

    python3 -m venv glider_dev

    It's important to keep "glider_dev" name

  2. Activate venv

    source glider_dev/bin/activate

  3. Install requirements

    pip install --upgrade google-api-python-client google-auth-httplib2 google-auth-oauthlib oauth2client

  4. See setup in Modules Documentation to know how to install specific requirements.

  5. Activate env variables. You should create two .env files

    In this case, the variables for any submodule are the same.

    a) .env.dev for developing and testing

    b) .env.prod for production You can use either of them, as the variables are the same.

    ENVIRONMENT=<Development/Production>
    AWS_KEY_ID=<YOUR_AWS_KEY_ID>
    AWS_SECRET_KEY=<YOUR_AWS_SECRET_KEY>
    REGION=<YOUR_AWS_REGION>
    FTP_ACCESS=<YOUR_MONGO_COLLECTION_FOR_FTP_CREDENTIALS>

  6. Save and run, remember that you can use either of them

    source deploy/dev_vars.sh for developing env

    source deploy/prod_vars.sh for production env

Testing

Local Testing

  1. Activate venv and variables

    source glider_dev/bin/activate

    source deploy/dev_vars.sh

  2. Get any link to shared report or folder of reports. See How to get a google Drive shared link

  3. Into tests/ create a python script following this format: test_gdrive.py

    import sys
    sys.path.insert(1, "src/upload_module/gdrive")
    sys.path.insert(1, "run")
    from FileCleaning import run_file_cleaning
    from SendNotification import send_uploader_notification
    input_data = {
    "module": "drive",
    "drive_link": <<shared_gdrive_link>>,
    "bucket": <<s3 bucket where files will be uploaded>>,
    "path": <<s3 path where files will be uploaded>>,
    "file": "",
    "on_finish": <<webhook.url>>
    }
    if __name__ == "__main__":
    try:
    run_file_cleaning(input_data)
    send_uploader_notification("success", input_data["module"], input_data["on_finish"])
    except Exception as e:
    message = str(sys.exc_info()[2]) + str(traceback.format_exc())
    send_uploader_notification("fail", input_data["module"], input_data["on_finish"], message)

  4. Run script

    python tests/test_gdrive.py

    Google Drive Submodule

    It receives a google drive link, downloads reports in a temporal folder to applies the Cleaning Procedure

    Workflow

    Input format

    The input for this module consists in a json object which detemines what is upload method, it could be via Google Drive or manually, In this case, these are parameters used for Google Drive method

    Field Description Example
    module This field determines which route will be called. In this case it must to be "drive" or "google" "drive"
    statement_id This is used to identify the statement run where the uploaded files will be processed. This field is optional and can be discarded. "statement_run_78"
    drive_link Link to google drive file/folder "https://drive.google.com/drive/"
    client_id String which represents the id of each client "client1"
    bucket String which contains the input bucket (where files will be uploaded) this item must to exist in your AWS account "bucket-input"
    path String which contains the input path (where files will be uploaded) this item must to exist in your AWS account "path-input"
    file List of files to upload. This field is required just when module = "manually", in any other case, this files must to be an empty list []
    on_finish Webhook link to send final status https://your_webhook.site/

    Local Example

    Example using python

    1. Set testing variables

    2. Run server locally a) If you are using flask, run

      python app.py

      b) If you use Docker, run

      docker-compose -f local.yml up

      It runs a local sever in 5000 port. If you go to http://127.0.0.1:5000 in your browser you would see the api interface

    3. Get any link to shared report or folder of reports. See How to get a google Drive shared link

    1. Into tests/ create a python script following this format: test_route_uploads.py

      import os
      import sys
      import traceback
      import requests
      if __name__ == "__main__":
      input_data = {
      "module": "drive",
      "client_id": <<client_id>>,
      "statement_id": <<statement_id>>, #Optional
      "drive_link": <<shared_gdrive_link>>,
      "bucket": <<s3 bucket where files will be uploaded>>,
      "path": <<s3 path where files will be uploaded>>,
      "file": ""/[],
      "on_finish": <<webhook.url>>
      }
      response = requests.post('http://127.0.0.1:5000/uploads', json=input_data)
      print(response.json())

    2. Run script

      python tests/test_route_uploads.py

      If everything is ok, it returns the following:

      {
      'date': CURRENT_DATE,
      'message': 'Module procedure will start immediately',
      'module': 'uploads',
      'status': 200
      }

      If a field is missing or it's wrong, it returns:

      {
      'date': CURRENT_DATE,
      'error_in_field': ['client_id'],
      'error_message': 'Field required',
      'module': 'uploads',
      'status': 500
      }
