Uploads Modules Workflow

Description

Uploads module is composed of these submodules:

  • Google Drive
  • Upload File Manually
  • FTP

Each module has their own features (explained in each section) but in at general performance, all of them do the following:

  1. Receives reports
  2. Decompress gzip, zip files
  3. Skips non txt, csv, xls, tsv files
  4. Normalizes filename (replaces blank spaces by underscores)
  5. Fits csv, xls reports to csv standard
  6. Uploads reports to s3 The 2-5 steps will called **"Cleaning Procedure"**

Google Drive Submodule

It receives a google drive link, downloads reports in a temporal folder to applies the Cleaning Procedure

Workflow

How to create venv dev

  1. Create a new virtual env (if you have created this venv pass to step 4)

python3 -m venv glider_dev

It's important to keep "glider_dev" name

  1. Activate venv

source glider_dev/bin/activate

  1. Install requirements

pip install --upgrade google-api-python-client google-auth-httplib2 google-auth-oauthlib oauth2client

  1. See setup in Modules Documentation to know how to install specific requirements.

  2. Activate env variables. You should create two .env files

    In this case, the variables for any submodule are the same.

    a) .env.dev for developing and testing

    b) .env.prod for production You can use either of them, as the variables are the same.

    ENVIRONMENT=<Development/Production>
    AWS_KEY_ID=<YOUR_AWS_KEY_ID>
    AWS_SECRET_KEY=<YOUR_AWS_SECRET_KEY>
    REGION=<YOUR_AWS_REGION>
    FTP_ACCESS=<YOUR_MONGO_COLLECTION_FOR_FTP_CREDENTIALS>
  3. Save and run, remember that you can use either of them

source deploy/dev_vars.sh for developing env

source deploy/prod_vars.sh for production env

Testing

Local Testing

  1. Activate venv and variables

source glider_dev/bin/activate

source deploy/dev_vars.sh

  1. Get any link to shared report or folder of reports. See How to get a google Drive shared link
  2. Into tests/ create a python script following this format: test_gdrive.py
import sys
sys.path.insert(1, "src/upload_module/gdrive")
sys.path.insert(1, "run")
from FileCleaning import run_file_cleaning
from SendNotification import send_uploader_notification
input_data = {
"module": "drive",
"drive_link": <<shared_gdrive_link>>,
"bucket": <<s3 bucket where files will be uploaded>>,
"path": <<s3 path where files will be uploaded>>,
"file": "",
"on_finish": <<webhook.url>>
}
if __name__ == "__main__":
try:
run_file_cleaning(input_data)
send_uploader_notification("success", input_data["module"], input_data["on_finish"])
except Exception as e:
message = str(sys.exc_info()[2]) + str(traceback.format_exc())
send_uploader_notification("fail", input_data["module"], input_data["on_finish"], message)
  1. Run script

python tests/test_gdrive.py

AWS Testing

When infrastructure is deployed in AWS:

  1. Go to AWS -> Step Functions
  2. Select the step function (select_and_execute_module)
  3. Click on Start execution

(Optional: set a testing name)

  1. Set json object
{
"module": "drive",
"drive_link": <<shared_gdrive_link>>,
"bucket": <<s3 bucket where files will be uploaded>>,
"path": <<s3 path where files will be uploaded>>,
"file": "",
"on_finish": <<webhook.url>>
}
  1. Click on Start execution. If the execution is successfully, you can see it in EC2 instance
ssh -i Glider-Postgres.pem ubuntu@ec2-35-179-24-180.eu-west-2.compute.amazonaws.com
