AWS infrastructure

We use ECS (Fargate) which contains several routes (one per module). Each module should be executed according to input parameters and the route called by FE.
Fargate is a serverless service, so when the modules are called, it are executed via Docker container.
This container is saved in a private repository in ECR and used via ECS tasks. More information in OTTO: AWS Fargate Installation

How to deploy code

Follow this steps to get and deploy code

  1. Open a terminal and clone repository
    cd ~
    git clone https://github.com/OTTOGlider/glider.git

  2. Go to glider folder

    cd glider

  3. Install requirements

    pip install -r src/importer/requirements.txt

  4. Set env variables. You should create two .env files

    a) .env.dev for developing and testing
    b) .env.prod for production
    Both files have the same format, just add your dev and prod credentials for each file. Here's an example

    ENVIRONMENT=<STAGING/PRODUCTION>
    AWS_KEY_ID=<YOUR_AWS_KEY_ID>
    AWS_KEY_SECRET=<YOUR_AWS_SECRET_KEY>
    MONGO_GLIDER=<MONGO_CONNECTION_LINK>
    DATABASE_URL=<POSTGRES_DATABASE>
    DB=<YOUR_DB>
    COLLECTION=<YOUR_MONGO_COLLECTION>
    TRENDS_COLLECTION=<YOUR_MONGO_COLLECTION_FOR_TRENDS>
    SNAPSHOTS_DB=<YOUR_MONGO_DB_FOR_SNAPSHOTS>
    FORMATS=<YOUR_MONGO_COLLECTION_FOR_FORMATS>
    SNAPSHOTS=<YOUR_MONGO_COLLECTION_FOR_SNAPSHOTS>
    CAT_COLLECTION=<YOUR_MONGO_COLLECTION_FOR_CATALOGUE>
    REGION=<YOUR_AWS_REGION>
    AS_COLLECTION=<YOUR_MONGO_COLLECTION_FOR_AS_CATALOGUE>

  5. Save and run

    cp .env.dev .env for developing env

    cp .env.prod .env for production env

    It loads env vars to be used still you change them.

How to deploy code in docker

  1. Create a Dockerfile (if it doesn't exist) and copy this format

    # Use oficial Python image
    FROM python:3.10
    # Set /app as workspace
    WORKDIR /app
    # Copy requirements to container
    COPY requirements.txt .
    # Copy module files to container
    COPY src/importer /app/src/importer
    COPY src/upload_module /app/src/upload_module
    COPY src/report_generation /app/src/report_generation
    COPY src/statement_generation /app/src/statement_generation
    COPY module_executions /app/module_executions
    COPY run /app/run
    # Create a venv
    RUN python -m venv /glider
    ENV PATH="/glider/bin:$PATH"
    # Install TeX Live
    RUN apt-get update && apt-get install -y texlive texlive-latex-extra \
    gnumeric \
    && rm -rf /var/lib/apt/lists/*
    # Install dependencies
    RUN pip install --upgrade pip
    RUN pip install -r requirements.txt
    # Copy app to container
    COPY app.py .
    # Expose 5000 port
    EXPOSE 5000
    # Start app when container is running
    CMD ["python", "app.py"]

  2. Build the container, in this case, it will be called just "glider"

    docker build -t glider .

    Optional: Add a tag to container

    docker tag glider:latest

  3. Run container

    docker run -p 5000:5000 glider

    It runs a local sever in 5000 port. If you request a GET to http://127.0.0.1:5000 you would have this response

    {
    "date": CURRENT_DATE,
    "env": CURRENT_ENV,
    "message": "Welcome to Glider Project",
    "status": 200
    }
