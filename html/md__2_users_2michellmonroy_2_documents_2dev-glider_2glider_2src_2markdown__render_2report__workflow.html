Report Module Workflow

Description

Generates a csv file which contains all the processed lines of an specific period/month.

The columns considered to build the csv report are the following:

quantity isrc_id
release_id release_title
track_title artists
label_id territory_code
types service_id
date_str accounting_date
foreign_currency total_foreign
exchange_rate local_currency
total_local

The general performance is the following:

  1. Select just the required fields/columns from MongoDB
  2. Extract only columns shown previously
  3. Generate csv
  4. Compress csv to gzip file
  5. Sent to user

Also this module is able to generate unmatched lines report. This feature is possible only when client has a catalogue with us.

Workflow

How to create venv dev

  1. Create a new virtual env (if you have created this venv pass to step 4)

    python3 -m venv glider_dev

  2. Activate venv

    It's important to keep "glider_dev" name

    source glider_dev/bin/activate

  3. Install requirements (only in case that you haven't install requirements yet)

    pip install -r src/importer/requirements.txt

  4. Activate env variables. You should create two .env files

    In this case, the variables for any submodule are the same.

    a) .env.dev for developing and testing

    b) .env.prod for production

    ENVIRONMENT=<Development/Production>
    AWS_KEY_ID=<YOUR_AWS_KEY_ID>
    AWS_SECRET_KEY=<YOUR_AWS_SECRET_KEY>
    SENDER_EMAIL="example@example.me"
    SENDER_PASSWORD=<YOUR_SENDER_PASSWORD>
    MONGO_GLIDER=<MONGO_CONNECTION_LINK>
    DATABASE_URL=POSTGRES_DATABASE
    DB=<YOUR_DB>
    COLLECTION=<YOUR_MONGO_COLLECTION>
    SNAPSHOTS_DB=<YOUR_MONGO_DB_FOR_SNAPSHOTS>
    SNAPSHOTS=<YOUR_MONGO_COLLECTION_FOR_SNAPSHOTS>
    CAT_COLLECTION=<YOUR_MONGO_COLLECTION_FOR_CATALOGUE>
    REGION=<YOUR_AWS_REGION>
    AS_COLLECTION=<YOUR_MONGO_COLLECTION_FOR_AS_CATALOGUE>
    LAMBDA_FUNCTION=<LAMBDA_FUNCTION_TO_GENERATE_PRESIGNED_LINK>
    INSTANCE_ID_LITE=<EC2_TO_EXECUTE_LITE_PROCEDURE>
    INSTANCE_ID_PRO=<EC2_TO_EXECUTE_PRO_PROCEDURE>

  5. Save and run, remember that you can use either of them

    source deploy/dev_vars.sh for developing env

    source deploy/prod_vars.sh for production env

Testing

Local Testing

  1. Activate venv and variables

    source glider_dev/bin/activate

    source deploy/dev_vars.sh

  2. Make sure you have at less one processed file in MongoDB. If don't, use importer module to process it

  3. Into tests/ create a python script following this format: test_reports.py

    import os
    import sys
    import traceback
    sys.path.insert(1, 'src/report_generation')
    sys.path.insert(1, 'run')
    from GenReport import build_csv
    from SendNotifications import send_report_notification
    input_data = {
    "module": "reporting",
    "tag": <<"test"/"prod">>,
    "date": <<accounting date>>,
    "client_id": <<client id>>,
    "bucket": <<s3 bucket where files will be uploaded>>,
    "path": <<s3 path where files will be uploaded>>,
    "report_type": <<"monthly"/"unmatch">>,
    "client_email": <<user@email.example>>,
    "on_finish": <<webhook.url>>
    }
    if __name__ == "__main__":
    try:
    build_csv(input_data)
    send_report_notification("success", input_data["module"], input_data["on_finish"], input_data["client_email"])
    except Exception as e:
    message = str(sys.exc_info()[2]) + str(traceback.format_exc())
    send_report_notification("fail", input_data["module"], input_data["on_finish"], input_data["client_email"], message)

  4. Run script

    python tests/test_reports.py

AWS Testing

When infrastructure is deployed in AWS:

  1. Go to AWS -> Step Functions
  2. Select the step function (select_and_execute_module)

  3. Click on Start execution

    (Optional: set a testing name)

  4. Set json object

    {
    "module": "reporting",
    "tag": <<"test"/"prod">>,
    "date": <<accounting date>>,
    "client_id": <<client id>>,
    "bucket": <<s3 bucket where files will be uploaded>>,
    "path": <<s3 path where files will be uploaded>>,
    "report_type": <<"monthly"/"unmatch">>,
    "client_email": <<user@email.example>>,
    "on_finish": <<webhook.url>>
    }

  5. Click on Start execution. If the execution is successfully, you can see it in EC2 instance

    ssh -i Glider-Postgres.pem ubuntu@ec2-35-179-24-180.eu-west-2.compute.amazonaws.com
