Statements Module Workflow

Description

Generates a csv and pdf file which contains all the processed lines of one or more specifics artist/label.

This feature is possible only when client has a catalogue with us.

The columns considered to build the csv and pdf reports are in our database already (result of importer processing, include cataloge matching process) and are the following:

Statement Field Data type Database Field
Sales Period Date date
Sales Territory String territory_code
DSP Name String service_id
Product Type String type
Barcode String release_id
ISRC String isrc_id
Label String label_id
Catalogue Number String generated after cat matching
Artist String artists
Title String track_title
Quantity Integer quantity
Total Local String total_local
Label Royalty Percent String Assign by client using OAS
Royalty Due String Calculated by this module

The general performance is the following:

  1. Select just the required fields/columns from MongoDB
  2. Extract only columns shown previously
  3. Generate csv
  4. Compress csv to gzip file
  5. Search client/organization info (Royalty percentage)
  6. Generate PDF file
  7. Sent to user

Workflow

Input format

The input for this module consists in a json object which detemines what are the fields to be shown in the statement, the client/organization to be used for the royalty percentage and the artists/labels used to generate the statement

Field Description Example
module This field determines which route will be called. In this case it must to be "statement" "statement", "statements"
tag This field determines which env variables have to be loaded "test", "testing", "prod"
select_by Set the field (column) to be used as filter, these filters should be values availables in the database. "service_id", "artists", "label_id"
field_value It's a list of the values to filter and generate the statement using just these value. It generates on csv and one pdf for each value in field_value, but it shoudl be under the same field, i.e just artists, just labels, etc. ["Taylor", "Sam", "The Ramones"]
client_id String which represents the id of each client "client1"
bucket String which contains the input bucket (where files will be uploaded) this item must to exist in your AWS account "bucket-input"
path String which contains the input path (where files will be uploaded) this item must to exist in your AWS account "path-input"

client_email

Email which client will be receive the notification about his statement status

clien.nosp@m.t@ex.nosp@m.ample.nosp@m..com

on_finish

Webhook link to send final status

https://your_webhook.site/

Local Example

Example using python

  1. Make sure that you have been processed at less one file using importer including the cataogue matching procedure
  2. Set testing variables

  3. Run server locally a) If you are using flask, run

    python app.py

    b) If you use Docker, run

    docker run -p 5000:5000 glider

    It runs a local sever in 5000 port. If you go to http://127.0.0.1:5000 in your browser you would see the api interface

  4. Into tests/ create a python script following this format: test_statements.py

    import os
    import sys
    import traceback
    import requests
    if __name__ == "__main__":
    input_data = {
    "module": "statement",
    "tag": <<"test"/"prod">>,
    "select_by": <<field to filter>>,
    "field_value": <<list of value(s)>>
    "client_id": <<client id>>,
    "bucket": <<s3 bucket where files will be uploaded>>,
    "path": <<s3 path where files will be uploaded>>,
    "client_email": <<user@email.example>>,
    "on_finish": <<webhook.url>>
    }
    response = requests.post('http://127.0.0.1:5000/statements', json=input_data)
    print(response.json())

    This script will send a POST request to the server with the data you want to test. The response will be printed in the console.

  5. Run script

    python tests/test_statements.py

    If everything is ok, it returns the following:

    {
    'date': CURRENT_DATE,
    'message': 'Module procedure will start immediately',
    'module': 'statements',
    'status': 200
    }

    If a field is missing or it's wrong, it returns:

    {
    'date': CURRENT_DATE,
    'error_in_field': ['client_id'],
    'error_message': 'Field required',
    'module': 'statements',
    'status': 500
    }
