How to Deploy AWS Infrastructure

Prerequisites

  • Python >= 3.9
  • AWS Account
  • Terraform install

How to deploy terraform code

Note: There exists a file called "main.tf" it contains entire infrastructure and it NEVER WOULD BE CHANGED because source code is connected to it.
Any type of editing in this file could cause the entire project to stop working as intended.

  1. Open a terminal in the glider project and put it on infrastructure folder.

    cd infrastructure/

  2. In Glider/infrastructure there are main.tf and variables.tf files. This is AWS infrastructure and env variables.

    The env variables format is the following configure it with your AWS ACCESS KEY ID and AWS SECRET ACCESS KEY (You can copy these values from .env file)

    variable "zones" {
    type = string
    default = <<aws_zone>>
    }
    variable "access_key"{
    type = string
    default = <<your_aws_access_key_id>>
    }
    variable "secret_key"{
    type = string
    default = <<your_aws_secret_access_key>>
    }
    variable "ip_ec2_lite"{
    type = string
    default = <<your_aws_ip_ec2_lite>>
    }
    variable "ip_ec2_pro"{
    type = string
    default = <<your_aws_ip_ec2_pro>>
    }

  3. Start terraform env using:

    terraform init

  4. Create a new env to testing or dev:

    terraform workspace new <<env_name>>

    This command helps us to have a version control.

  5. Verify that you are in the correct env using:

    terraform workspace list

    It should show the current workspace.

  6. To check if the code in main.tf is right use:

    terraform plan

  7. If everything is ok, apply infrastructure use:

    terraform apply

    Wait for it.

  8. Go to AWS and confirm that entire configuration is created successfully (lambda and step functions).

WARNING: To destroy infrastructure, use terraform destroy. This destroy all infrastructure indicated in main.tf
